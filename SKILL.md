# 公众号文章写作 Skill

## 核心理念

- **写作引擎可替换**：正文写作模块默认使用通用骨架。用户指定「晚点风格」时，自动切换为 `latepost-writer` Skill 的六步写作流程。
- **一次触发，完整交付**：正文写完后，自动产出标题备选、CTA、朋友圈文案，并完成排版注入，直到发布 Checklist 生成为止。
- **不猜测，主动确认**：信息不足时优先向用户提问，不自行假设。

---

## ⚠️ 务必先读：内容安全红线

微信公众号审核严厉，以下内容类型**禁止**撰写或必须在写作前彻底去敏：

| 🚫 绝对禁止 | 示例 | 替代方案 |
|------------|------|---------|
| 中美政治/外交叙事 | "政治表演""城下之盟""卡脖子" | 去掉所有政治定性，只保留商业事实 |
| 芯片制裁/科技管制博弈 | "中国被卡脖子""美国打压" | 转为纯技术/商业角度：国产替代进展、算力效率优化 |
| 引用美国批评中国的内容 | "美国强硬派说……" | 删除或替换为中立第三方观点 |
| 时政人物直接评价 | 特朗普/习近平决策分析 | 只陈述客观事件，不评价值得/不值得 |
| 军事/安全相关 | "军事禁用""安全风险" | 完全删除 |

> 💡 **黄金法则**：如果文章能让读者读出「站在某一方批评另一方」，这篇文章就不能发。WeChat 审核不是审查事实，是审查**叙事角度**。

**安全选题公式**：纯商业 + 纯技术 + 纯行业趋势。不碰地缘政治、不做价值判断、不引用对立观点。

**过往踩线案例**：见 `references/content-safety-cases.md`（含 H200 芯片文章被废弃的完整复盘）。

## 八步标准流程

### STEP 0：内容安全审查（新增，最高优先级）

**在 STEP 1 确认参数之前**，先对选题进行安全审查：

1. 扫描素材中是否含有上述 🚫 禁止类型
2. 如发现敏感内容，**主动向用户预警**：明确指出哪些角度会踩线
3. 提供安全替代方案（如：从「中美芯片博弈」转为「国产替代技术进展」）
4. **用户确认替代方案后再进入 STEP 1**

> ⚠️ 不要等用户自己发现。主动预警是你的责任。

---

### STEP 1：确认写文参数

主动询问用户以下信息（全部确认后再进入 STEP 2）：

```
请帮我确认以下写文参数：

1. 【主题】：这篇文章写关于什么？（公司/产品/行业/事件/人物）
2. 【写作风格】：通用模式 or 晚点LatePost深度报道模式？（默认：通用）
3. 【排版参考账号】：模仿哪个公众号的视觉风格？（默认：第101个机器人）
4. 【核心观点Thesis】：这篇文章最想说的一句话是什么？有独特角度吗？
5. 【目标字数】：长文（3000字+）/ 中文（1500字左右）/ 短评（1000字内）
```

参数收集完毕后，给用户一份确认摘要，等待「确认」后进入 STEP 2。

---

### STEP 2：选择内容骨架

根据 STEP 1 参数，从以下骨架库中匹配：

| 骨架类型 | 适用场景 | 资源文件 |
|---------|---------|---------|
| 战略拆解型 | 公司发展路径/品牌战略全解读 | `resources/playbook_strategy.md` |
| 人物特写型 | 创始人/高管深度人物稿 | `resources/playbook_profile.md` |
| 行业变局型 | 赛道趋势/市场格局分析 | `resources/playbook_industry.md` |
| 产品逻辑型 | 产品深度拆解/用户体验分析 | `resources/playbook_product.md` |
| 创投生态型 | 融资/资本/投资逻辑叙事 | `resources/playbook_capital.md` |

输出：推荐骨架 + 说明理由，等用户确认。

> **若用户选择晚点风格**：从 STEP 3 起，调用 `latepost-writer` Skill 的写作逻辑（STEP 3 采集、STEP 4 写作），完成后回到本 Skill 的 STEP 4（内容包）继续执行。

---

### STEP 3：全网素材采集 + 正文写作

**素材采集（至少覆盖以下渠道）**：
- 官方渠道：官网、公告、投资者关系页
- 权威媒体：36氪、虎嗅、界面新闻、财经、晚点LatePost
- 数据报告：IDC、艾瑞、欧睿、招股书/财报
- 社区讨论：知乎、小红书、微博
- 海外视角：英文媒体、Twitter/X、Reddit

**正文写作规范**（通用模式）：

**语气铁律（最高优先级 — 用户 477 反复强调）**：
- ✅ **纯观察叙事（记者口吻）**：陈述事实、描述现象、引述数据，像记者写稿
- ❌ 禁止教育/导师口吻：不用"你应该""企业需要""建议""启示""结论"
- ❌ 禁止说教结尾：不总结教训、不给人生启发、不升华价值观
- 用户评语：「太浅了太教育了写观察就好」

**排版风格**：
- 短段 + 留白，Notion / Linear 风格
- 用 `· · ·` 分隔符（而非序号章节标题）
- 无首行缩进，段落短小（手机端友好）
- 关键句加粗强调
- 数据必须有来源支撑，不捏造数据
- 尾部：`-end-` + `文 | 技能101`

---

### STEP 4：生成配套内容包

正文完成后，**立即输出**以下配套内容，无需用户再次触发：

#### 📌 标题备选（3个）

分别对应三种不同的读者钩子策略：

```
【悬念型】（制造认知冲突，让人想点进来）
→ 示例：追觅，为什么主动放弃价格战？

【数据型】（信息密度高，可信度强）
→ 示例：150亿营收、30国第一，追觅用8年做成了什么

【情绪型】（共鸣/惊叹/忧虑，触发情感反应）
→ 示例：追觅跑得太快了，快到有点让人不安
```

#### 📌 CTA 话术

置于文章结尾，2-3句，引导关注/转发/留言：

```
写法参考：
- 语气与账号风格一致（第101个机器人：理性+科技感+有温度）
- 提出一个思考问题引导留言
- 结尾引导关注或转发
```

#### 📌 朋友圈转发文案

50-80字，可直接复制使用：

```
写法要求：
- 第一句提炼核心观点（带数据或强结论）
- 第二句说明为什么值得读
- 结尾：「👆 点击阅读全文」或「链接在评论区」
```

---

### STEP 4.5：封面图生成（必做）

公众号文章**必须配封面图**（用户会索要）。在 STEP 4 完成后、STEP 5 排版前主动生成。

**路径选择**：

| 条件 | 方法 | 说明 |
|------|------|------|
| `OPENAI_API_KEY` 可用 | `imagegen` CLI (`gpt-image-2`) | 推荐，质量最高。尺寸用 `1536×656`（接近 2.35:1） |
| API key 不可用 | HTML + Playwright 截图 | 可靠备选，本 session 验证通过 |

**HTML+Playwright 封面图流程**（详见 `references/cover-html-workflow.md`）：
1. 写 HTML（900×383 px，含标题/副标题/视觉装饰）
2. Playwright screenshot → PNG
3. 插入飞书文档

> ⚠️ 不要等用户提醒才做封面图。写完正文后直接生成。

### STEP 5：排版交付

1. 将正文 Markdown 通过 JavaScript 注入 `https://md.doocs.org`
2. 截图确认右侧渲染效果
3. 通知用户：

```
✅ 排版已就绪！

你的浏览器中已打开 md.doocs.org，文章内容已加载。

操作步骤：
1. 打开浏览器 → 找到 md.doocs.org 标签页
2. 点击右上角「复制」按钮
3. 进入微信公众号后台 → 新建图文消息 → Ctrl+V 粘贴
4. 手动将 01～0X 小标题颜色改为橙色（#FF6600）
```

---

### STEP 6：飞书文档交付（优先，默认执行）

写完正文 + 封面图后，**先交付飞书文档给用户审阅**，不直接推公众号。

1. 用 `lark-cli docs +create --title "文章标题" --markdown @./article.md --as bot`
2. 用 `lark-cli docs +media-insert --doc "$DOC_ID" --file "./cover.png" --as bot` 插入封面图
3. 用 `lark-cli api POST` 授权用户 `full_access`
4. 返回飞书链接

### STEP 6.5：推送草稿到公众号（md2wechat）

> ⚠️ **CLI 版本注意**：当前安装的是旧版 md2wechat（路径 `/Users/a477/Library/Python/3.9/bin/md2wechat`），**不支持** `inspect`/`preview`/`convert` 子命令。直接使用旧版 flag 语法。

**凭证**：需设置 `WECHAT_APPID` 和 `WECHAT_APP_SECRET`。AppID 为 `wxaad976676ad2d32c`（公众号「技能101」）。AppSecret 通常存储在 `/tmp/.env`，如丢失需向用户索取。

**推送命令**（旧版 CLI）：

```bash
M2W="/Users/a477/Library/Python/3.9/bin/md2wechat"
$M2W --markdown /tmp/article.md \
  --title "标题（≤32字）" \
  --cover /tmp/cover.png \
  --author "技能101" \
  --comment
```

**参数说明**：
- `--markdown`：文章 Markdown 文件路径
- `--title`：标题，**≤32 中文字符**（WeChat 限制）
- `--cover`：本地封面图路径（**必传**，否则报 `MISSING_COVER_IMAGE`）
- `--author`：作者名（≤16 字符）
- `--comment`：开启留言（**必带**）
- `--html`：可选，用 HTML 模式推送以保排版（绿色主题不稳定勿用）
- `--summary`：可选，手动设置摘要

**常见错误**：
- `MISSING_COVER_IMAGE`：未传 `--cover` 或封面图路径无效
- `WECHAT_APPID not found`：环境变量未设置，需 `source /tmp/.env` 或手动 export
- 标题过长：WeChat 限制约 64 字节，中文一个字 3 字节，控制在 20 字以内最安全
